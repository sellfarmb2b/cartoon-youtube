# 앱 테스트 가이드

## 빌드 완료 확인

터미널에서 다음 명령어로 빌드 결과를 확인하세요:

```bash
ls -la dist/
```

다음과 같은 파일이 생성되어야 합니다:
- **macOS**: `dist/유튜브_영상_생성기.app`
- **Windows**: `dist/유튜브_영상_생성기.exe`
- **Linux**: `dist/유튜브_영상_생성기`

## 테스트 방법

### 방법 1: Finder에서 실행 (macOS)

1. **Finder 열기**
   ```bash
   open dist/
   ```

2. **앱 더블클릭**
   - `유튜브_영상_생성기.app` 파일을 더블클릭합니다
   - 첫 실행 시 보안 경고가 나타날 수 있습니다

3. **보안 경고 해결 (macOS)**
   - "확인되지 않은 개발자" 경고가 나타나면:
     - 시스템 설정 → 개인 정보 보호 및 보안 → "열기" 버튼 클릭
   - 또는 터미널에서:
     ```bash
     xattr -cr dist/유튜브_영상_생성기.app
     ```

### 방법 2: 터미널에서 실행 (macOS)

```bash
cd "/Users/rohbyoungkon/Desktop/유튜브 프로젝트"
open dist/유튜브_영상_생성기.app
```

또는 직접 실행:

```bash
dist/유튜브_영상_생성기.app/Contents/MacOS/유튜브_영상_생성기
```

### 방법 3: Python 스크립트로 테스트

간단한 테스트 스크립트를 실행:

```bash
python -c "
import subprocess
import time
import webbrowser
import os

# 앱 실행
app_path = 'dist/유튜브_영상_생성기.app/Contents/MacOS/유튜브_영상_생성기'
if os.path.exists(app_path):
    process = subprocess.Popen([app_path])
    time.sleep(3)
    webbrowser.open('http://127.0.0.1:5001')
    print('앱이 실행되었습니다. 브라우저에서 확인하세요.')
else:
    print('앱을 찾을 수 없습니다.')
"
```

## 기대 동작

1. **앱 시작**
   - 앱이 실행되면 자동으로 브라우저가 열립니다
   - 브라우저가 열리지 않으면 `http://127.0.0.1:5001` (또는 다른 포트)에 수동으로 접속

2. **서버 확인**
   - 터미널에 "유튜브 영상 생성기 서버 시작" 메시지가 표시됩니다
   - 포트 번호가 표시됩니다

3. **기능 테스트**
   - 대본 입력
   - 프롬프트 생성
   - 이미지 생성
   - 영상 생성

## 문제 해결

### 앱이 실행되지 않음

1. **권한 확인**
   ```bash
   chmod +x dist/유튜브_영상_생성기.app/Contents/MacOS/유튜브_영상_생성기
   ```

2. **로그 확인**
   - 터미널에서 실행하여 오류 메시지 확인

### 포트 충돌

- 앱이 자동으로 다른 포트(5002, 5003 등)를 찾아 사용합니다
- 브라우저에서 여러 포트를 시도해보세요

### FFmpeg 오류

- FFmpeg가 설치되어 있는지 확인:
  ```bash
  ffmpeg -version
  ```
- PATH에 FFmpeg가 있는지 확인

### 리소스 파일 누락

- `templates/` 폴더와 `static/` 폴더가 앱에 포함되었는지 확인
- 빌드 시 경고 메시지 확인
